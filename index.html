JavaScript Variables: var, let, const

What's the Difference?

In JavaScript, you can declare variables in three ways. Here's the simple explanation:

var - The old way (avoid in new code)

let - Use when the value will change

const - Use when the value won't change

var - The Old Way

var was used in older JavaScript code. It has some problems:

var name = "John"; var name = "Jane"; // This works - confusing!
Problem: You can declare the same variable multiple times, which can cause bugs.

let - For Changing Values

Use let when you need to change the value later:

let age = 25; age = 26; // This works let age = 30; // Error! Can't redeclare
Good uses for let:
- Counter in loops
- User input values
- Anything that changes

const - For Constant Values

Use const for values that shouldn't change:

const PI = 3.14159; const API_URL = "https://api.example.com"; PI = 3.14; // Error! Can't change const
Note: With objects and arrays, you can't reassign the variable, but you can change the content inside.
const person = { name: "John" }; person.name = "Jane"; // This works person = { name: "Mike" }; // Error!

Simple Rule to Remember

Start with const - If you need to change it later, switch to let.
Avoid var - Use let and const instead.
Good Practice:
// Use const by default const MAX_USERS = 100; const SITE_NAME = "My App"; // Use let only when needed let currentUsers = 0; let isLoggedIn = false;

Scope Differences

var is function-scoped - available in the whole function

let & const are block-scoped - available only inside { } blocks

if (true) { var x = 10; // Available outside let y = 20; // Only available inside } console.log(x); // Works console.log(y); // Error!
